FTSO.eu uses third party Availibility Zone which consists of one or more data centers with redundant power, networl, and connectivity in European region.

We use Availibility Zones to run applications and databases in production enviroments with high availability, fault tolerance and scalability that would otherwise be impossible to achieve within single data center.

All Availability Zones in the region are interconnected via high-bandwidth, low-latency network over a fully redundant dedicated metro fiber that distributes high-throughput, low-latency networks between them.

All traffic between Availability Zones is encrypted. Network performance is sufficient to achieve synchronous replication between Availibility Zones.

Availability Zones make partitioning of applications for high availability very simple.

Partitioning an application into different zones enables companies to be isolated and protects them from problems such as blackout, lightning, tornadoes, eathquake and more.

Availability Zones are physically separated from each other by a significant distance of many kilometers while remaining within 100km of each other.
